Sistema de Gestión de Productos y Usuarios

Desarrollé una plataforma administrativa completa en Laravel diseñada para optimizar la gestión de productos, usuarios y permisos dentro de una aplicación empresarial. El sistema implementa un control de acceso basado en roles (RBAC) mediante el paquete Spatie Laravel Permission, garantizando una administración segura, escalable y flexible. Además, cuenta con una interfaz moderna, responsive y amigable, construida con TailwindCSS y componentes dinámicos de Blade.

Sistema de Envío de Correos Automatizado

🎯 Objetivo y problema resuelto

El objetivo principal del proyecto fue crear una solución integral de administración que permitiera a las empresas o equipos técnicos gestionar sus productos, usuarios y roles de forma eficiente y centralizada. Antes, la gestión de permisos y usuarios se realizaba de forma manual y poco estructurada, lo que generaba errores de acceso, duplicidad de datos y falta de trazabilidad. Con esta plataforma, se resolvió ese problema implementando un sistema automatizado de control de acceso, donde cada usuario tiene permisos asignados según su rol, reduciendo riesgos y mejorando la productividad operativa.

⚙️ Tecnologías y herramientas

  • Laravel 10 (PHP)
  • Laravel Breeze
  • Blade Templates + TailwindCSS
  • MySQL
  • Composer y NPM
  • Git / GitHub

📈 Resultados medibles

  • Implementación exitosa de un sistema de roles y permisos totalmente dinámico, que permite escalar el control de acceso sin modificar código.
  • Reducción del tiempo de gestión de usuarios y permisos en más del 60% comparado con un flujo manual.
  • Interfaz intuitiva y responsiva, optimizada para escritorio y dispositivos móviles.
  • Arquitectura modular que facilita la extensión del sistema con nuevos módulos (por ejemplo, reportes, facturación o inventario avanzado).

Sistema de Envío de Correos Automatizado

🎯 Objetivo y problema resuelto

Automatizar el envío de correos electrónicos transaccionales y personalizados, resolviendo la necesidad de gestionar fácilmente plantillas y mensajes desde un backend centralizado.

⚙️ Tecnologías y herramientas

  • FastAPI (Python)
  • SendGrid API
  • PostgreSQL + SQLAlchemy
  • Jinja2, OAuth2/JWT, Swagger UI

📈 Resultados medibles

  • Reducción del tiempo de envío masivo en un 70 %.
  • CRUD completo para plantillas HTML reutilizables.
  • Tasa de entrega del 95 % y latencia promedio menor a 1 s.
  • Arquitectura escalable lista para integración con microservicios.
